## 1. AdSense Script — Quy tắc placement

### Loader script (`adsbygoogle.js`)

Google chỉ generate code với attribute `async`. Không có `defer`, không có `type="module"`. Đây là form chính thức duy nhất:

```html
<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXXXXXXXXXXXXXX"
     crossorigin="anonymous"></script>
```

- **`async` là bắt buộc**, không phải tùy chọn — nó là cơ chế loading của AdSense, không phải performance optimization.
- **Không thêm `defer`** bên cạnh `async`: `defer` không có tác dụng khi `async` đã có mặt (browser bỏ qua `defer` nếu có `async`). Về mặt kỹ thuật không vỡ, nhưng không phải code Google generate nên tiềm ẩn rủi ro ToS.
- **Không dùng `type="module"`**: module scripts luôn deferred theo spec HTML, làm xáo trộn thứ tự load của adsbygoogle.

### Inline push script

```html
<script>
  (adsbygoogle = window.adsbygoogle || []).push({});
</script>
```

- Inline script **không thể có `defer` hay `async`** — browser không apply hai attribute này cho inline scripts (chỉ dành cho external scripts).
- Inline script **không thể có `type="module"`** — module scope sẽ làm `window.adsbygoogle` không truy cập được đúng cách.
- Push script phải nằm **ngay sau** thẻ `<ins>` tương ứng để đảm bảo thứ tự DOM.

### `<ins class="adsbygoogle">` — static hay inject?

- **Static HTML là bắt buộc cho AdSense crawler** (Google-Display-Ads-Bot dùng khi verify site khi apply): crawler này **không execute JavaScript**, nó đọc HTML thô. Nếu `<ins>` chỉ tồn tại sau JS execution, bot sẽ không thấy ad slot và có thể reject.
- Sau khi site được approve, Mediapartners-Google (crawler ad relevance) có thể xử lý JS ở mức nào đó, nhưng Google không cam kết điều này.
- **Quy tắc an toàn**: `<ins>` phải có trong initial HTML (server-rendered / pre-rendered). JS có thể post-process (lazy load sau user interaction) nhưng DOM element phải tồn tại trước.
- Kỹ thuật delay load chấp nhận được: inject lại `src` attribute vào loader script sau user interaction (pattern `data-delay`), **không phải** inject `<ins>` element từ đầu.

### Googlebot vs AdSense crawler

- **Googlebot (Search indexing)**: dùng Web Rendering Service (WRS), có execute JavaScript giống modern browser.
- **Google-Display-Ads-Bot (AdSense approval verification)**: đây là bot riêng, **không execute JavaScript** — nó verify site structure khi bạn submit domain vào AdSense.
- **Mediapartners-Google (Ad relevance)**: crawl sau khi approved, share cache với Googlebot, có khả năng xử lý JS ở mức hạn chế.
- Kết luận: pre-rendered static HTML bảo vệ cả 2 trường hợp quan trọng nhất (approval + relevance).

**Nguồn:** [Google AdSense Help — Where to place AdSense code](https://support.google.com/adsense/answer/9274516), [Google AdSense Help — Ad unit code placement](https://support.google.com/adsense/answer/9190028), [Publisher Ads Audits — async-ad-tags](https://developers.google.com/publisher-ads-audits/reference/audits/async-ad-tags), [About the AdSense ads crawler](https://support.google.com/adsense/answer/99376)

**Kết luận cho render.js:**

1. `render.js` phải output `<ins class="adsbygoogle">` và inline push script trong static HTML output — **không inject bằng JS sau khi load**.
2. Loader script (`adsbygoogle.js`) đặt trong `<head>` với `async` duy nhất — không thêm attribute nào khác.
3. Thứ tự cố định trong mỗi page: `<head>` chứa loader script → `<body>` chứa 3 cụm `<ins>` + push script tại vị trí ad-top, ad-mid, ad-bottom.
4. Hard rule cho `config.js`: tất cả script khác dùng `defer` hoặc `type="module"`, **ngoại lệ duy nhất** là AdSense loader dùng `async`.

## 2. hreflang — 18 locales

### Ba phương pháp implementation

| Phương pháp | Phù hợp với |
|---|---|
| HTML `<link>` tags trong `<head>` | Sites nhỏ, dễ control per-page |
| XML sitemap với `<xhtml:link>` | **Sites lớn, static generated — phương pháp tốt nhất** |
| HTTP headers `Link:` | Files không phải HTML (PDF, v.v.) |

Google coi 3 phương pháp là **tương đương**. Với 18 locales × N articles, sitemap method là duy nhất scalable vì không bloat `<head>` với 18+ `<link>` tags mỗi page.

### Cú pháp sitemap với hreflang

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <url>
    <loc>https://example.com/en/article-slug/</loc>
    <xhtml:link rel="alternate" hreflang="en" href="https://example.com/en/article-slug/"/>
    <xhtml:link rel="alternate" hreflang="vi" href="https://example.com/vi/article-slug/"/>
    <xhtml:link rel="alternate" hreflang="fr" href="https://example.com/fr/article-slug/"/>
    <!-- ... tất cả 18 locales ... -->
    <xhtml:link rel="alternate" hreflang="x-default" href="https://example.com/en/article-slug/"/>
  </url>
</urlset>
```

**Lưu ý namespace**: phải khai báo `xmlns:xhtml` trong thẻ `<urlset>`.

### x-default — khi nào và trỏ về đâu

- `x-default` là fallback cho user có ngôn ngữ **không nằm trong 18 locales** của site.
- Trỏ về **URL của version ngôn ngữ mặc định** (thường là `en`) — không phải trang language selector (trừ khi bạn có trang auto-redirect).
- Phải **nhất quán**: tất cả các page trong cluster đều dùng cùng một URL pattern cho x-default.
- Chỉ **một x-default tag** mỗi URL.

### Return tag — yêu cầu bắt buộc

Đây là lỗi phổ biến nhất (75% implementations có lỗi): **mọi page phải tham chiếu ngược lại tất cả các pages khác**. Nếu `en` liệt kê `vi`, `fr`, v.v. thì `vi` và `fr` cũng phải liệt kê `en` và nhau.

Trong sitemap method: vì mỗi `<url>` block liệt kê **tất cả** 18 alternate URLs (kể cả self-reference), yêu cầu này được thỏa mãn tự động nếu implement đúng.

### Common mistakes gây hreflang bị ignore

1. **Missing self-referencing tag**: mỗi URL phải có `<xhtml:link>` trỏ về chính nó.
2. **Sai ISO code**: dùng `UK` thay vì `GB`; dùng `zh-CN` cho Hong Kong thay vì `zh-HK`.
3. **Relative URLs**: phải dùng absolute URL (`https://...`), không phải `/path/`.
4. **Broken bidirectional links**: nếu thiếu return tags, Google ignore toàn bộ cluster.
5. **Hai locales trỏ cùng URL**: mỗi hreflang phải chỉ đến một URL duy nhất, khác nhau.

**Nguồn:** [Google Search Central — Localized Versions](https://developers.google.com/search/docs/specialty/international/localized-versions), [Backlinko — hreflang guide](https://backlinko.com/hreflang-tag), [International Web Mastery — x-default](https://internationalwebmastery.com/blog/mastering-hreflang-x-default/)

**Kết luận cho seo.js / sitemap (feed.js):**

1. **Không dùng HTML `<link>` method** cho 18 locales — sẽ thêm 18-19 dòng vào mỗi `<head>`, bloat HTML không cần thiết. Dùng **sitemap method** hoàn toàn.
2. `seo.js` không cần generate hreflang `<link>` tags trong HTML output — chỉ cần canonical URL là đủ trong `<head>`.
3. `feed.js` (sitemap generator) phải: (a) khai báo `xmlns:xhtml` namespace; (b) trong mỗi `<url>` block, thêm self-referencing + tất cả 17 alternate locales + x-default; (c) x-default = English locale URL.
4. Language codes phải theo ISO 639-1 (ví dụ: `vi`, `en`, `fr`, `de`) hoặc kết hợp với region khi cần (`zh-TW`, `zh-CN`, `pt-BR`).
5. Sitemap phải regenerate toàn bộ khi publish bài mới (do slug immutable, cần thêm entries mới, không cần update cũ).

## 3. AI Content + AdSense Policy

### AI content có được phép không?

**Được phép**, với điều kiện. Google không reject site vì dùng AI — họ reject vì **low-quality content**. Chính sách tập trung vào chất lượng đầu ra, không phải quy trình tạo ra.

### Điều kiện để AI content pass

- **Human oversight bắt buộc**: ai đó phải review output AI trước khi publish.
- **Unique insights**: content phải có thông tin/góc nhìn mà người đọc không tìm thấy ở nơi khác.
- **E-E-A-T signals**: Experience, Expertise, Authoritativeness, Trustworthiness — cụ thể là: author attribution, About page rõ ràng, credentials hiển thị.
- **Không dùng hedge language**: tránh lạm dụng "may," "could," "possibly" — dấu hiệu AI draft chưa được edit.

### "Low-value content" — định nghĩa cụ thể

Reviewer flag content có các đặc điểm:

- Cấu trúc lặp lại giống hệt nhau trên mọi trang (templated structure).
- Claim không cụ thể — không có dữ liệu, ví dụ, hay kinh nghiệm thực tế.
- Không có author với verifiable background.
- Content đọc như thể có thể xuất hiện trên bất kỳ competitor site nào.

### Publishing velocity — mức độ rủi ro

- **Không có official limit** từ Google về số bài/ngày.
- **Thực tế quan sát (March 2026 Core Update)**: sites publish 50+ bài AI/ngày **không có human oversight** bị penalty.
- Rủi ro tăng khi: volume cao + quality thấp + apply AdSense ngay sau khi spam content.
- **50–900 bài/ngày của Akao CMS** nằm trong vùng rủi ro cao nếu không có quality signals rõ ràng.

### Không cần disclose AI

AdSense ToS (tính đến 2026) không yêu cầu disclosed AI usage. Tuy nhiên, transparent editorial oversight **tăng** E-E-A-T.

**Nguồn:** [AdSense Audit — AI Content Policy 2026](https://adsenseaudit.net/guides/adsense-ai-content-policy-2026), [TheHumanizeAI — AdSense AI Policy 2026](https://thehumanizeai.pro/articles/google-adsense-ai-content-policy-2026), [HasteWire — Is AI Content Allowed in AdSense](https://hastewire.com/blog/is-ai-content-allowed-in-google-adsense-full-guide)

**Kết luận cho pipeline:**

1. Pipeline cần có **quality gate** trước khi publish: tối thiểu phải check word count, unique entities, non-templated structure.
2. Mỗi bài phải có **author field** trong frontmatter — dù là bút danh, phải nhất quán và có trang profile tương ứng.
3. **Không publish bare AI draft**: pipeline cần enrichment step (thêm data cụ thể, thêm ví dụ, rework introduction và conclusion).
4. `config.js` cần có `publishVelocity` cap và `qualityMinScore` field để kiểm soát pipeline — không phải hard limit của Google nhưng là risk management.
5. Khi apply AdSense: không apply ngay khi vừa có content — cần ít nhất 2-4 tuần content history với engagement signals trước.

## 4. Sitemap Index — Scale lớn

### Khi nào cần sitemap index vs single sitemap

| Điều kiện | Giải pháp |
|---|---|
| Dưới 50,000 URLs và dưới 50MB | Single sitemap đủ |
| Trên 50,000 URLs HOẶC trên 50MB | **Bắt buộc dùng sitemap index** |

Với 18 locales × N articles: nếu N > ~2,700 articles (50,000 / 18 ≈ 2,778), cần sitemap index. Akao CMS sẽ vượt ngưỡng này nhanh.

### Format XML chính xác

```xml
<?xml version="1.0" encoding="UTF-8"?>
<sitemapindex xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <sitemap>
    <loc>https://example.com/sitemaps/sitemap-en.xml</loc>
    <lastmod>2026-06-08</lastmod>
  </sitemap>
  <sitemap>
    <loc>https://example.com/sitemaps/sitemap-vi.xml</loc>
    <lastmod>2026-06-08</lastmod>
  </sitemap>
  <!-- ... 16 locales còn lại ... -->
</sitemapindex>
```

- `<lastmod>` optional nhưng khuyến nghị — dùng W3C Datetime format (`YYYY-MM-DD` là đủ).
- Sitemap index file **tự nó cũng bị giới hạn** 50,000 `<loc>` entries (tức là có thể reference 50,000 child sitemaps).
- Google Search Console cho phép submit **tối đa 500 sitemap index files** mỗi site.

### Cấu trúc cho 18 locales × nhiều articles

**Chiến lược khuyến nghị — sitemap per locale:**

```
/sitemap-index.xml          ← submit cái này lên GSC
/sitemaps/sitemap-en.xml    ← tối đa 50,000 URLs tiếng Anh
/sitemaps/sitemap-vi.xml
/sitemaps/sitemap-fr.xml
...
/sitemaps/sitemap-{locale}.xml  (18 files)
```

Nếu một locale vượt 50,000 bài, split tiếp:

```
/sitemaps/sitemap-en-1.xml
/sitemaps/sitemap-en-2.xml
```

**Quy tắc hierarchy**: các child sitemaps phải nằm cùng directory hoặc **thấp hơn** trong hierarchy so với sitemap index. Không thể đặt index ở `/sitemap-index.xml` và reference child ở `https://otherdomain.com/...`.

### Tích hợp hreflang vào sitemap

Mỗi child sitemap (`sitemap-{locale}.xml`) chứa URLs của locale đó, và trong mỗi `<url>` block có đầy đủ `<xhtml:link>` cho tất cả 18 locales. Đây là cách Google khuyến nghị:

```xml
<!-- trong sitemap-en.xml -->
<url>
  <loc>https://example.com/en/slug/</loc>
  <xhtml:link rel="alternate" hreflang="en" href="https://example.com/en/slug/"/>
  <xhtml:link rel="alternate" hreflang="vi" href="https://example.com/vi/slug/"/>
  <!-- ... 16 locales khác ... -->
  <xhtml:link rel="alternate" hreflang="x-default" href="https://example.com/en/slug/"/>
</url>
```

**Lưu ý**: cùng bài đó cũng xuất hiện trong `sitemap-vi.xml` với tất cả alternate links — tức là mỗi bài xuất hiện 18 lần trong hệ thống sitemap.

**Nguồn:** [Google Search Central — Large Sitemaps](https://developers.google.com/search/docs/crawling-indexing/sitemaps/large-sitemaps), [Sitemaps.org FAQ](https://www.sitemaps.org/faq.html), [GTechMe — Multilingual XML Sitemaps](https://www.gtechme.com/insights/best-practices-for-multi-language-and-multi-region-xml-sitemaps-hreflang-support/)

**Kết luận cho feed.js:**

1. `feed.js` cần generate **sitemap index** từ đầu, không phải single sitemap — cấu trúc `/sitemap-index.xml` + `/sitemaps/sitemap-{locale}.xml` × 18.
2. Mỗi locale sitemap phải include namespace `xmlns:xhtml` và full `<xhtml:link>` cho tất cả 18 locales trong mỗi `<url>` block.
3. `lastmod` trong sitemap index = ngày publish bài mới nhất của locale đó (không phải ngày hôm nay cứng trong code).
4. Build step: regenerate only affected locale sitemaps khi có bài mới, sau đó regenerate sitemap index — không cần rebuild toàn bộ 18 sitemaps mỗi khi có 1 bài mới.
5. Path convention trong sitemap phải khớp với URL structure thực tế: nếu URL là `/vi/slug/` thì `<loc>` phải là `https://domain.com/vi/slug/`.

## 5. AdSense Approval Requirements

### Content requirements

- **Minimum articles**: không có official number nhưng consensus từ publishers là **15–25 bài chất lượng** trước khi apply. Ít hơn 10 bài gần như chắc bị reject.
- **Word count**: 800–1,500+ words mỗi bài, well-structured với headings, bullet points, nội dung thực sự hữu ích.
- **Niche consistency**: tránh mix quá nhiều topic không liên quan — focused niche được approve nhanh hơn.
- **Unique content**: không copy, không thin AI spam, không rewrite từ sites khác.

### Required pages (thiếu = instant rejection)

| Page | Yêu cầu |
|---|---|
| **About / About Us** | Giải thích site làm gì, ai điều hành |
| **Privacy Policy** | Nêu rõ data collection practices, AdSense cookies |
| **Contact Us** | Form hoặc email thực |
| **Terms & Conditions** | Khuyến nghị thêm |

### Technical requirements

- **HTTPS bắt buộc** — site không có SSL bị reject ngay lập tức.
- **Mobile responsive** — phải hiển thị tốt trên mobile (hơn 60% traffic là mobile).
- **Site speed** — dưới 3 giây load time.
- **XML sitemap** — phải có và submit lên Google Search Console.
- **Google Search Console** — site phải được index trong GSC trước khi apply.
- **ads.txt** — không bắt buộc để apply nhưng bắt buộc ngay sau khi được approve để tránh serving bị block.

### OG tags và structured data

Không có official requirement cụ thể từ AdSense về OG tags. Tuy nhiên:
- OG tags (`og:title`, `og:description`, `og:image`) giúp AdSense bot hiểu content context → better ad relevance → better CPM.
- `og:type`, `article:published_time` giúp Google phân loại content đúng.
- Schema.org `Article` structured data tăng E-E-A-T signals.

### Navigation requirements

- Site phải có **clear navigation structure** — homepage có links đến categories/articles.
- Breadcrumbs khuyến nghị cho sites với nhiều levels.
- Footer phải có links đến About, Privacy Policy, Contact.

### Traffic requirements

Không có official minimum. Tuy nhiên có **Google traffic** (từ Search) trước khi apply tạo thêm trust signal.

**Nguồn:** [Google AdSense Eligibility Requirements](https://support.google.com/adsense/answer/9724), [WebtimizeSolutions — AdSense Approval Guide 2026](https://webtimizesolutions.com/blogs/google-adsense-approval-guide-2026-complete-genuine-updated-information/), [AdPushup — AdSense Approval Strategies](https://www.adpushup.com/blog/google-adsense-approval/)

**Checklist trước khi apply AdSense:**

- [ ] Site có HTTPS (SSL certificate active)
- [ ] Tối thiểu 15-25 bài chất lượng (800+ words, unique, không phải thin AI)
- [ ] Page: About Us (giải thích site, team/author)
- [ ] Page: Privacy Policy (đề cập AdSense cookies và data collection)
- [ ] Page: Contact Us (email thực hoặc contact form)
- [ ] Page: Terms & Conditions
- [ ] Navigation rõ ràng — homepage → categories → articles
- [ ] Footer chứa links đến About, Privacy, Contact
- [ ] Site đã được index trên Google Search (kiểm tra qua GSC)
- [ ] XML sitemap đã được submit lên Google Search Console
- [ ] Site mobile responsive (test qua Google Mobile-Friendly Test)
- [ ] Page speed dưới 3 giây (test qua PageSpeed Insights)
- [ ] `ads.txt` file tạo ngay sau khi approved: `google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0`
- [ ] Không có prohibited content (bạo lực, adult, copyright vi phạm)
- [ ] Không có broken links trên navigation chính

## Tóm tắt — Những thứ PHẢI implement đúng ngay từ đầu

- **AdSense `<ins>` phải ở trong static HTML output** (`render.js`): Google-Display-Ads-Bot không execute JS khi verify site. Ba ad slots (ad-top, ad-mid, ad-bottom) phải pre-rendered. Loader script: `async` attribute duy nhất, đặt trong `<head>`. Inline push script: không có attribute, ngay sau `<ins>`.

- **Sitemap index từ đầu, không retrofit sau** (`feed.js`): với 18 locales, sẽ vượt 50,000 URLs nhanh. Cấu trúc `/sitemap-index.xml` + `/sitemaps/sitemap-{locale}.xml` × 18. Mỗi `<url>` trong child sitemap phải có đầy đủ `<xhtml:link>` cho tất cả 18 locales + x-default (trỏ về English). Không dùng HTML `<link>` hreflang — quá bloated cho 18 locales.

- **AI content pipeline cần quality gate không thể bỏ qua**: 50+ bài/ngày không human oversight = risk bị penalty và reject AdSense. Pipeline phải có enrichment step (unique data, specific claims), author attribution, và quality score minimum trước khi publish. Apply AdSense sau ít nhất 2–4 tuần với content history chất lượng.

- **Required pages phải được build trước khi apply AdSense**: About, Privacy Policy (đề cập AdSense cookies), Contact, Terms. Thiếu bất kỳ page nào = rejection tức thì. Privacy Policy đặc biệt quan trọng vì AdSense yêu cầu disclose cookie usage.

- **Slug immutable + URL structure nhất quán là nền tảng của hreflang**: mọi hreflang cluster reference đến absolute URLs. Nếu slug thay đổi, toàn bộ cluster bị broken. Thiết kế URL pattern ngay từ đầu (`/{locale}/{slug}/`) và không thay đổi — đây không phải constraint của AdSense riêng mà là yêu cầu kỹ thuật của cả SEO multilingual lẫn AdSense serving.
